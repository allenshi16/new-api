## 1. 系统架构概览

### 1.1 项目结构

```
new-api/
├── router/           # HTTP 路由定义
├── controller/       # 请求处理器（API 接口）
├── service/          # 业务逻辑层
├── model/            # 数据模型和数据库操作（GORM）
├── relay/            # AI API 转发核心
│   └── channel/      # 渠道适配器（OpenAI, Claude, Gemini 等）
├── middleware/       # 中间件（认证、限流、日志）
├── monitor/          # 监控模块（健康检查、告警）
├── setting/          # 配置管理
│   ├── ratio_setting/     # 模型定价比率
│   ├── operation_setting/ # 运营配置
│   └── system_setting/    # 系统配置
├── common/           # 通用工具（JSON、加密、Redis）
├── constant/         # 常量定义
├── dto/              # 数据传输对象
├── types/            # 类型定义
├── oauth/            # OAuth 认证实现
├── i18n/             # 后端国际化
└── web/              # React 前端
    └── src/
        ├── i18n/     # 前端国际化
        └── components/
```

### 1.2 核心模块职责

| 模块 | 职责 |
|------|------|
| `relay/channel/` | 各 AI 供应商的 API 适配器，处理请求转换和响应解析 |
| `monitor/` | 渠道健康监控、用户额度预警、过期提醒 |
| `setting/ratio_setting/` | 模型价格比率配置，决定 API 调用计费 |
| `setting/operation_setting/` | 运营相关配置（支付、令牌、监控等） |
| `model/` | 数据库模型定义和 CRUD 操作 |
| `oauth/` | GitHub、Discord、OIDC 等 OAuth 登录 |

### 1.3 数据流

```
用户请求 → Router → Middleware（认证/限流） → Controller → Service
    → Relay（选择渠道） → Channel Adaptor（转换请求） → 上游 AI API
    → Response（转换响应） → 用户
```

## 2. 部署指南

### 2.1 环境要求

| 组件 | 版本要求 |
|------|----------|
| Go | 1.22+ |
| Node.js / Bun | Node 18+ 或 Bun 1.0+ |
| 数据库 | SQLite（默认）/ MySQL 5.7.8+ / PostgreSQL 9.6+ |
| Redis | 6.0+（可选，推荐生产环境使用） |

### 2.2 快速启动

```bash
# 克隆代码
git clone https://github.com/your-org/new-api.git
cd new-api

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 启动后端
go run main.go

# 或编译后运行
go build -o nexus-api .
./nexus-api
```

### 2.3 前端构建

```bash
cd web
bun install
bun run build   # 生产构建
bun run dev     # 开发模式
```

### 2.4 Docker 部署（推荐）

```bash
# 构建镜像
docker build -t nexus-api:latest .

# 运行容器
docker run -d \
  --name nexus-api \
  -p 3000:3000 \
  -v /data/nexus-api:/app/data \
  -e SQL_DSN="sqlite:///app/data/nexus.db" \
  -e SESSION_SECRET="your-secret-key" \
  nexus-api:latest
```

### 2.5 生产环境部署建议

1. **数据库**: 使用 MySQL 或 PostgreSQL，而非 SQLite
2. **缓存**: 启用 Redis 提升性能
3. **反向代理**: 使用 Nginx/Caddy 处理 SSL 和静态文件
4. **监控**: 配置日志收集和健康监控
5. **备份**: 定期备份数据库

## 3. 环境变量配置

### 3.1 核心配置

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `PORT` | 服务端口 | 3000 | 否 |
| `SESSION_SECRET` | 会话密钥 | - | **是** |
| `CRYPTO_SECRET` | 加密密钥 | SESSION_SECRET | 否 |
| `SQL_DSN` | 数据库连接字符串 | SQLite | 否 |
| `REDIS_CONN_STRING` | Redis 连接字符串 | - | 否 |

### 3.2 数据库配置

```bash
# SQLite（默认）
SQLITE_PATH=/path/to/nexus.db

# MySQL
SQL_DSN=user:password@tcp(127.0.0.1:3306)/nexus?parseTime=true

# PostgreSQL
SQL_DSN=postgres://user:password@localhost:5432/nexus?sslmode=disable

# 日志数据库（可选，分离日志）
LOG_SQL_DSN=user:password@tcp(127.0.0.1:3306)/nexus_log?parseTime=true
```

### 3.3 Redis 配置

```bash
# 基础配置
REDIS_CONN_STRING=redis://localhost:6379/0
REDIS_POOL_SIZE=10

# 哨兵模式
REDIS_CONN_STRING=redis+sentinel://localhost:26379/mymaster/0
REDIS_MASTER_NAME=mymaster
REDIS_PASSWORD=yourpassword
```

### 3.4 监控配置

```bash
# 健康监控频率（秒）
CHANNEL_TEST_FREQUENCY=300

# 指标监控
ENABLE_METRIC=true
METRIC_SUCCESS_RATE_THRESHOLD=0.8  # 成功率阈值
AUTOMATIC_DISABLE_CHANNEL_ENABLED=true  # 自动禁用失败渠道

# 过期监控
EXPIRATION_MONITOR_FREQUENCY=3600
```

### 3.5 限流配置

```bash
# API 限流
GLOBAL_API_RATE_LIMIT_ENABLE=true
GLOBAL_API_RATE_LIMIT_NUM=1000
GLOBAL_API_RATE_LIMIT_DURATION=60

# Web 限流
GLOBAL_WEB_RATE_LIMIT_ENABLE=true
GLOBAL_WEB_RATE_LIMIT_NUM=500
GLOBAL_WEB_RATE_LIMIT_DURATION=60

# 关键接口限流
CRITICAL_RATE_LIMIT_ENABLE=true
CRITICAL_RATE_LIMIT_NUM=100
CRITICAL_RATE_LIMIT_DURATION=60
```

### 3.6 超时配置

```bash
# 请求超时
RELAY_TIMEOUT=300          # 总超时（秒）
STREAMING_TIMEOUT=300      # 流式响应超时
USER_CONTENT_REQUEST_TIMEOUT=30
```

### 3.7 安全配置

```bash
# TLS
TLS_INSECURE_SKIP_VERIFY=false

# SSRF 防护（通过管理后台配置）
# fetch_setting.enable_ssrf_protection: true
# fetch_setting.allowed_ports: ["80", "443", "8080", "8443"]

# 可信任重定向域名
TRUSTED_REDIRECT_DOMAINS=yourdomain.com,sub.yourdomain.com
```

## 4. 数据库管理

### 4.1 数据库类型选择

| 类型 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| SQLite | 开发/小型部署 | 零配置、单文件 | 不适合高并发 |
| MySQL | 生产环境 | 高性能、广泛支持 | 需单独部署 |
| PostgreSQL | 生产环境 | 功能丰富、扩展性强 | 需单独部署 |

### 4.2 数据库切换

切换数据库只需修改 `SQL_DSN` 环境变量，重启服务即可。

```bash
# 从 SQLite 切换到 MySQL
# 1. 导出 SQLite 数据
sqlite3 nexus.db .dump > backup.sql

# 2. 创建 MySQL 数据库
mysql -u root -p -e "CREATE DATABASE nexus CHARACTER SET utf8mb4;"

# 3. 导入数据（需转换语法）
# 4. 配置 SQL_DSN 并重启
```

### 4.3 备份策略

```bash
# SQLite 备份
sqlite3 nexus.db ".backup 'backup_$(date +%Y%m%d).db'"

# MySQL 备份
mysqldump -u user -p nexus > nexus_backup_$(date +%Y%m%d).sql

# PostgreSQL 备份
pg_dump -U user nexus > nexus_backup_$(date +%Y%m%d).sql
```

### 4.4 常用 SQL 命令

```sql
-- 查看所有用户
SELECT id, username, email, role, status, quota FROM users;

-- 查看所有渠道
SELECT id, name, type, status, base_url, models FROM channels;

-- 查看活跃令牌
SELECT id, name, key, status, remain_quota FROM tokens WHERE status = 1;

-- 查看 abilities 表（模型-渠道映射）
SELECT * FROM abilities WHERE channel_id = 3;

-- 重置用户密码（需先生成 bcrypt hash）
UPDATE users SET password = '$2a$10$...' WHERE username = 'admin';

-- 修复 abilities 表（清除后重建）
DELETE FROM abilities WHERE channel_id = 3;
-- 然后通过管理后台重新编辑渠道触发重建
```

### 4.5 数据库连接池配置

```bash
SQL_MAX_IDLE_CONNS=100      # 最大空闲连接
SQL_MAX_OPEN_CONNS=1000     # 最大打开连接
SQL_MAX_LIFETIME=60         # 连接最大生命周期（秒）
SQLITE_BUSY_TIMEOUT=5000    # SQLite 繁忙等待超时（毫秒）
```

## 6. 监控与告警

### 6.1 监控模块概览

| 文件 | 功能 |
|------|------|
| `monitor/health.go` | 渠道健康检查，自动禁用失败渠道 |
| `monitor/expiration.go` | 用户到期预警、额度不足提醒 |
| `monitor/metric.go` | 渠道成功率统计，触发自动禁用 |
| `monitor/channel.go` | 渠道状态变更通知（邮件） |
| `monitor/manage.go` | 监控策略判断逻辑 |

### 6.2 健康监控配置

```bash
# 健康检查频率（秒）
CHANNEL_TEST_FREQUENCY=300

# 自动禁用失败渠道
AUTOMATIC_DISABLE_CHANNEL_ENABLED=true

# 自动重新启用渠道
AUTOMATIC_ENABLE_CHANNEL_ENABLED=true
```

### 6.3 指标监控配置

```bash
# 启用指标监控
ENABLE_METRIC=true

# 成功率阈值（低于此值触发禁用）
METRIC_SUCCESS_RATE_THRESHOLD=0.8

# 队列大小
METRIC_QUEUE_SIZE=10000
```

### 6.4 过期与额度监控

```bash
# 过期检查频率（秒）
EXPIRATION_MONITOR_FREQUENCY=3600

# 低额度预警阈值（百分比）
QUOTA_WARNING_THRESHOLD=20
```

### 6.5 告警通知

系统通过邮件发送告警，需配置 SMTP：

```bash
# SMTP 配置（在管理后台设置）
SMTP_SERVER=smtp.example.com
SMTP_PORT=587
SMTP_USERNAME=alert@example.com
SMTP_PASSWORD=password
```

## 7. 渠道管理运维

### 7.1 渠道类型列表

渠道类型常量定义在 `constant/channel.go`：

| ID | 类型 | 说明 |
|----|------|------|
| 1 | OpenAI | 官方 OpenAI API |
| 14 | Anthropic | Claude API |
| 17 | Ali | 阿里通义千问 |
| 20 | OpenRouter | OpenRouter 聚合 |
| 24 | Gemini | Google Gemini |
| 43 | DeepSeek | DeepSeek API |
| 59 | Qiniu | 七牛云 AI（新增） |

### 7.2 添加新渠道步骤

1. **定义渠道类型常量** (`constant/channel.go`)
   ```go
   ChannelTypeNewProvider = 60  // 新渠道类型
   ```

2. **添加 BaseURL** (`constant/channel.go`)
   ```go
   "https://api.newprovider.com", // 60 - NewProvider
   ```

3. **添加 APIType 映射** (`common/api_type.go`)
   ```go
   case constant.ChannelTypeNewProvider:
       apiType = constant.APITypeOpenAI  // 或自定义
   ```

4. **创建适配器**（如需自定义模型列表）
   ```
   relay/channel/newprovider/
   ├── constant.go   # 模型列表
   └── adaptor.go    # 适配器实现
   ```

5. **注册适配器** (`relay/relay_adaptor.go`)
   ```go
   import "github.com/.../relay/channel/newprovider"
   
   case constant.APITypeNewProvider:
       return &newprovider.Adaptor{}
   ```

6. **前端渠道选项** (`web/src/constants/channel.constants.js`)
   ```js
   { value: 60, color: 'blue', label: 'NewProvider' }
   ```

7. **重新构建并重启**

### 7.3 渠道健康状态

渠道状态值：
- `1`: 已启用
- `2`: 已禁用（手动）
- `3`: 已禁用（自动）

```sql
-- 查看渠道健康状态
SELECT channel_id, success, latency, last_test_time 
FROM channel_health WHERE success = false;

-- 手动重新启用渠道
UPDATE channels SET status = 1 WHERE id = 3;
-- 同步 abilities 表
UPDATE abilities SET enabled = true WHERE channel_id = 3;
```

### 7.4 渠道故障处理流程

1. 检查健康监控日志
2. 手动测试渠道
3. 检查 API Key 有效性
4. 检查 BaseURL 配置
5. 检查上游供应商状态
6. 修复后重新启用

## 8. 常见运维操作

### 8.1 服务管理

```bash
# 启动服务
go run main.go
# 或
./nexus-api

# 停止服务
kill $(pgrep nexus-api)
# 或
fuser -k 3000/tcp

# 重启服务
kill $(pgrep nexus-api) && sleep 2 && ./nexus-api &

# 查看服务状态
curl http://localhost:3000/api/status
```

### 8.2 缓存管理

```bash
# 清除 Redis 缓存
redis-cli FLUSHDB

# 清除内存缓存（重启服务）
# 或通过管理后台操作
```

### 8.3 用户管理

```sql
-- 重置用户密码（使用 bcrypt）
-- 先生成 hash：go run -e 'package main...'

-- 查看用户详情
SELECT * FROM users WHERE username = 'admin';

-- 修改用户额度
UPDATE users SET quota = 100000000 WHERE id = 1;

-- 禁用用户
UPDATE users SET status = 2 WHERE id = 5;
```

### 8.4 令牌管理

```sql
-- 查看活跃令牌
SELECT id, name, key, remain_quota, unlimited_quota 
FROM tokens WHERE status = 1 AND user_id = 1;

-- 禁用令牌
UPDATE tokens SET status = 2 WHERE id = 10;

-- 设置令牌额度限制
UPDATE tokens SET remain_quota = 500000, unlimited_quota = false WHERE id = 10;
```

### 8.5 修复 Abilities 表

当修改渠道模型后，abilities 表可能需要手动同步：


或通过管理后台调用 `FixAbility` 功能。

### 8.6 日志查看

## 9. 故障排查指南

### 9.1 常见问题

#### 9.1.1 用户登录失败

**症状**: 提示"用户名密码不正确或用户已被封禁"

**排查步骤**:
1. 检查用户状态 `SELECT status FROM users WHERE username = 'admin'`
2. 检查密码哈希是否正确
3. 重置密码

#### 9.1.2 API 调用失败

**症状**: "no available channels for model xxx"

**排查步骤**:
1. 检查渠道是否启用
2. 检查渠道是否包含该模型
3. 检查 abilities 表是否有映射
4. 检查用户分组是否匹配

#### 9.1.3 渠道测试失败

**症状**: 400/401/403 错误

**排查步骤**:
1. 检查 API Key 是否有效
2. 检查 BaseURL 是否正确（注意 `/v1` 路径）
3. 检查模型名称格式是否匹配上游要求
4. 直接 curl 测试上游 API

### 9.2 诊断命令

```bash
# 检查服务状态
curl -s http://localhost:3000/api/status | jq '.data.system_name'

# 检查渠道模型列表
curl -s http://localhost:3000/api/models | jq '.data.59'

# 直接测试上游 API
curl -X POST "https://api.example.com/v1/chat/completions" \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"test-model","messages":[{"role":"user","content":"hi"}]}'
```

### 9.3 性能调优

```bash
# 增加数据库连接池
SQL_MAX_OPEN_CONNS=2000

# 启用 Redis 缓存
REDIS_CONN_STRING=redis://localhost:6379/0
REDIS_POOL_SIZE=50

# 调整限流配置
GLOBAL_API_RATE_LIMIT_NUM=2000
```

## 10. 安全配置

### 10.1 会话安全

```bash
# 强随机 SESSION_SECRET（生产必须）
SESSION_SECRET=$(openssl rand -hex 32)

# 会话过期时间（管理后台配置）
SESSION_EXPIRY_SECONDS=86400
```

### 10.2 API 安全

```bash
# 限流配置
GLOBAL_API_RATE_LIMIT_ENABLE=true
CRITICAL_RATE_LIMIT_ENABLE=true

# TLS 验证
TLS_INSECURE_SKIP_VERIFY=false  # 生产环境必须为 false
```

### 10.3 SSRF 防护

管理后台配置 `fetch_setting`:
- `enable_ssrf_protection`: true
- `allow_private_ip`: false
- `allowed_ports`: ["80", "443"]
- `domain_list`: 允许的域名白名单

### 10.4 2FA 配置

支持 Passkey/WebAuthn 和 TOTP：
- 管理后台 → 设置 → Passkey 配置
- 用户可自行启用 2FA
