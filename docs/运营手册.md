## 1. 定价机制

### 1.1 定价模型概述

Nexus AI 采用 **模型倍率定价** 机制，每个模型有对应的价格倍率，最终费用 = Token 数量 × 基准价格 × 模型倍率。

**基准价格定义** (`setting/ratio_setting/model_ratio.go`):
```
$0.002 = 1 quota（基准单位）
$1 = 500 quota
￥1 = 68.5 quota（汇率 7.3）
```

### 1.2 模型价格配置

价格配置文件位置：`setting/ratio_setting/model_ratio.go`

**典型模型倍率示例**：

| 模型 | 倍率 | 实际价格（$/1M tokens） |
|------|------|------------------------|
| gpt-4o | 1.25 | $2.5 |
| gpt-4o-mini | 0.075 | $0.15 |
| gpt-4-turbo | 5 | $10 |
| o1 | 7.5 | $15 |
| o1-mini | 0.55 | $1.1 |
| deepseek-v3 | 0.14 | $0.28 |
| claude-3-5-sonnet | 1.5 | $3 |

### 1.3 如何调整定价

**方式一：修改代码配置**

编辑 `setting/ratio_setting/model_ratio.go`，修改 `defaultModelRatio` 和 `defaultModelPrice`：

```go
var defaultModelRatio = map[string]float64{
    "gpt-4o": 1.25,  // 修改倍率
    "your-model": 2.0,  // 新增模型定价
}
```

**方式二：管理后台配置**

1. 登录管理后台
2. 进入 **设置** → **模型设置**
3. 配置模型价格比率

### 1.4 定价策略建议

| 场景 | 建议倍率 |
|------|----------|
| 旗舰模型（GPT-4o, Claude） | 1.0 - 2.0 |
| 高性价比模型（DeepSeek, Qwen） | 0.1 - 0.3 |
| 入门模型（gpt-4o-mini） | 0.05 - 0.1 |
| 图像生成模型 | 20 - 50 |
| 嵌入模型 | 0.01 - 0.05 |

### 1.5 充值金额定价

配置文件：`setting/operation_setting/payment_setting.go`

```go
AmountOptions:  []int{10, 20, 50, 100, 200, 500},  // 充值面额
AmountDiscount: map[int]float64{
    100: 0.9,   // 100元享9折
    200: 0.85,  // 200元享85折
    500: 0.8,   // 500元享8折
},
```

## 2. 支付配置

### 2.1 支付方式概览

| 支付方式 | 适用场景 | 配置文件 |
|----------|----------|----------|
| Stripe | 国际支付（信用卡） | `setting/payment_stripe.go` |
| Creem | 国际支付 | `controller/topup_creem.go` |
| Waffo | 国际支付 | `controller/topup_waffo.go` |
| 充值链接 | 自定义支付渠道 | 管理后台配置 |

### 2.2 Stripe 配置

**配置项** (`setting/payment_stripe.go`):

| 变量 | 说明 | 获取方式 |
|------|------|----------|
| StripeApiSecret | Stripe API 密钥 | Stripe Dashboard → API Keys |
| StripeWebhookSecret | Webhook 签名密钥 | Stripe Dashboard → Webhooks |
| StripePriceId | 价格 ID | Stripe Dashboard → Products |
| StripeUnitPrice | 单价（美元） | 默认 8.0 |
| StripeMinTopUp | 最小充值金额 | 默认 1 美元 |

**配置步骤**：

1. 注册 Stripe 账户：https://stripe.com
2. 创建产品和价格
3. 配置 Webhook URL：`https://your-domain/api/topup/notify`
4. 在管理后台填入 Stripe 配置

### 2.3 充值链接配置

适用于接入支付宝、微信等国内支付：

1. 进入管理后台 → **设置** → **系统设置**
2. 配置 `TopUpLink`（充值跳转链接）
3. 用户点击充值后跳转到外部支付页面
4. 支付成功后手动完成订单（管理员操作）

### 2.4 兑换码充值

用于活动赠送、批量分发：

```sql
-- 创建兑换码
INSERT INTO redemptions (id, key, quota, status, used_count, max_count)
VALUES (1, 'GIFT-XXXX-YYYY', 100000, 1, 0, 100);

-- 用户输入兑换码即可获得额度
```

## 3. 用户注册与登录

### 3.1 注册方式概览

| 方式 | 开关配置 | 适用场景 |
|------|----------|----------|
| 用户名密码注册 | PasswordRegisterEnabled | 通用注册 |
| GitHub OAuth | GitHubOAuth | 开发者社区 |
| Discord OAuth | DiscordOAuth | 游戏/社区 |
| OIDC | OIDCEnabled | 企业内部 |
| LinuxDO | LinuxDOOAuth | 技术社区 |
| 自定义 OAuth | CustomOAuthProvider | 企业定制 |

### 3.2 OAuth 配置

**GitHub OAuth 配置**：

1. GitHub → Settings → Developer settings → OAuth Apps
2. 创建新应用：
   - Application name: Nexus AI
   - Homepage URL: `https://your-domain`
   - Authorization callback URL: `https://your-domain/oauth/github/callback`
3. 获取 Client ID 和 Client Secret
4. 在管理后台填入配置

**Discord OAuth 配置**：

1. Discord Developer Portal → Applications
2. 创建新应用，配置 OAuth2
3. Redirect URL: `https://your-domain/oauth/discord/callback`
4. 在管理后台填入配置

**OIDC 配置**：

适用于企业内部系统（如 Keycloak）：

```bash
OIDC_ENABLED=true
OIDC_CLIENT_ID=your-client-id
OIDC_CLIENT_SECRET=your-secret
OIDC_AUTHORIZATION_ENDPOINT=https://auth.example.com/oauth2/auth
OIDC_TOKEN_ENDPOINT=https://auth.example.com/oauth2/token
OIDC_USER_INFO_ENDPOINT=https://auth.example.com/oauth2/userinfo
```

### 3.3 邮箱验证

配置邮箱验证可防止恶意注册：

```bash
EMAIL_VERIFICATION_ENABLED=true
```

**SMTP 配置**（管理后台）：
- SMTP 服务器
- SMTP 端口
- SMTP 用户名/密码
- 发件人邮箱

### 3.4 注册流程

**密码注册流程**：

```
用户填写信息 → 邮箱验证（如启用） → 创建账户 → 
生成默认令牌 → 分配默认分组 → 完成
```

**OAuth 注册流程**：

```
点击 OAuth 按钮 → 授权页面 → 回调 → 
获取用户信息 → 创建/绑定账户 → 完成
```

### 3.5 邀请机制

用户可通过邀请码邀请新用户：

- 每个用户有唯一的邀请码（AffCode）
- 新用户注册时可填写邀请码
- 邀请人可获得奖励额度

配置邀请奖励：
```sql
-- 设置邀请奖励额度
UPDATE options SET value = '50000' WHERE key = 'AffQuota';
```

## 4. 用户分组与权限

### 4.1 分组概念

用户分组用于控制用户可访问的模型和渠道：

| 分组 | 说明 | 典型用途 |
|------|------|----------|
| default | 默认分组 | 普通用户 |
| vip | VIP 分组 | 高级用户 |
| enterprise | 企业分组 | 企业客户 |
| auto | 自动分组 | 智能路由 |

### 4.2 创建分组

通过管理后台创建分组：

1. 进入 **渠道管理**
2. 编辑渠道的分组字段
3. 添加新分组名称（如 `vip`）
4. 配置该分组的可用模型

### 4.3 用户权限级别

| 角色 | 值 | 权限 |
|------|-----|------|
| 普通用户 | 1 | 仅 API 调用 |
| 管理员 | 10 | 管理后台部分权限 |
| 超级管理员 | 100 | 全部权限 |

### 4.4 分组模型配置

为不同分组配置可用模型：

```sql
-- 渠道配置多分组
UPDATE channels SET group = 'default,vip' WHERE id = 1;

-- VIP 分组用户可访问更多模型
-- 通过 abilities 表自动管理
```

### 4.5 自动分组（Auto Group）

智能路由功能，根据模型自动选择最佳分组：

配置自动分组：
```bash
DEFAULT_USE_AUTO_GROUP=true
```

## 5. 令牌管理

### 5.1 令牌概念

令牌（Token）是用户调用 API 的凭证：

| 字段 | 说明 |
|------|------|
| Key | API 密钥（sk-xxx） |
| RemainQuota | 剩余额度 |
| UnlimitedQuota | 无限额标识 |
| ExpiredTime | 过期时间 |
| Group | 所属分组 |
| ModelLimits | 模型限制 |

### 5.2 令牌创建流程

**用户自行创建**：

1. 登录用户面板
2. 进入 **令牌管理**
3. 点击创建令牌
4. 设置名称、额度、过期时间
5. 复制密钥保存

**管理员创建**：

```sql
-- 直接创建令牌
INSERT INTO tokens (user_id, key, name, status, remain_quota, group)
VALUES (1, 'sk-xxxxxxxxxxxx', '测试令牌', 1, 50000, 'default');
```

### 5.3 令牌额度管理

**增加额度**：
```sql
UPDATE tokens SET remain_quota = remain_quota + 50000 WHERE id = 10;
```

**设置无限额度**：
```sql
UPDATE tokens SET unlimited_quota = true WHERE id = 10;
```

### 5.4 令牌过期管理

```sql
-- 设置过期时间（Unix 时间戳）
UPDATE tokens SET expired_time = 1735689600 WHERE id = 10;

-- 永不过期
UPDATE tokens SET expired_time = -1 WHERE id = 10;
```

### 5.5 令牌安全建议

- 定期轮换令牌
- 设置合理的过期时间
- 使用 IP 白名单限制
- 为不同用途创建独立令牌
- 监控令牌使用量

## 6. 充值与余额

### 6.1 充值流程

```
用户选择金额 → 选择支付方式 → 支付 → 
系统确认 → 额度增加 → 完成
```

### 6.2 充值金额选项

默认选项：10、20、50、100、200、500（单位：额度）

修改充值选项：
```go
// setting/operation_setting/payment_setting.go
AmountOptions: []int{50, 100, 200, 500, 1000},
```

### 6.3 充值折扣策略

配置阶梯折扣：

```go
AmountDiscount: map[int]float64{
    100: 0.95,   // 100额度实付95%
    200: 0.90,   // 200额度实付90%
    500: 0.85,   // 500额度实付85%
    1000: 0.80,  // 1000额度实付80%
}
```

### 6.4 充值记录查询

```sql
-- 查看用户充值记录
SELECT id, user_id, amount, money, status, create_time 
FROM topups WHERE user_id = 1 ORDER BY create_time DESC;

-- 管理员查看全部充值
SELECT * FROM topups ORDER BY create_time DESC LIMIT 50;
```

### 6.5 手动完成订单

对于外部支付渠道，管理员可手动完成订单：

1. 进入管理后台 → **充值管理**
2. 找到待处理订单
3. 点击 **完成订单**
4. 用户额度自动增加

## 7. 渠道运营

### 7.1 渠道配置最佳实践

**多渠道备份**：

同一模型配置多个渠道，实现容灾：

```
模型: gpt-4o
├── 渠道1: OpenAI 官方（优先级高）
├── 渠道2: OpenRouter（备用）
└── 渠道3: Azure OpenAI（备用）
```

**渠道优先级配置**：

```sql
-- 设置优先级（数值越大优先级越高）
UPDATE channels SET priority = 100 WHERE id = 1;  -- 主渠道
UPDATE channels SET priority = 50 WHERE id = 2;   -- 备用渠道
```

### 7.2 渠道健康监控

系统自动监控渠道健康：

- 响应时间 > 30秒：标记为异常
- 连续失败：自动禁用（需开启配置）

配置自动禁用：
```bash
AUTOMATIC_DISABLE_CHANNEL_ENABLED=true
```

### 7.3 渠道测试

定期测试渠道可用性：

```bash
# 通过 API 测试
curl "http://localhost:3000/api/channel/test/1?model=gpt-4o"

# 批量测试所有渠道
curl "http://localhost:3000/api/channel/test/all"
```

### 7.4 渠道余额监控

部分渠道支持余额查询：

- OpenAI
- OpenRouter
- SiliconFlow
- DeepSeek

余额不足时系统会发送告警邮件。

### 7.5 渠道定价策略

根据上游成本调整倍率：

| 渠道类型 | 建议加价比例 |
|----------|-------------|
| OpenAI 官方 | 10% - 20% |
| OpenRouter | 5% - 10% |
| 国内供应商 | 0% - 5% |

## 9. 数据分析与报表

### 9.1 用户统计

```sql
-- 用户总数
SELECT COUNT(*) FROM users;

-- 今日新增用户
SELECT COUNT(*) FROM users WHERE created_time > UNIX_TIMESTAMP(CURDATE());

-- 活跃用户（最近7天有调用）
SELECT COUNT(DISTINCT user_id) FROM logs WHERE created_at > UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY));
```

### 9.2 渠道统计

```sql
-- 各渠道调用次数
SELECT channel_id, COUNT(*) as calls, SUM(quota) as cost
FROM logs GROUP BY channel_id ORDER BY calls DESC;

-- 渠道成功率
SELECT 
    channel_id,
    SUM(CASE WHEN success = 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as success_rate
FROM logs GROUP BY channel_id;
```

### 9.3 模型统计

```sql
-- 各模型调用次数
SELECT model_name, COUNT(*) as calls, SUM(quota) as cost
FROM logs GROUP BY model_name ORDER BY calls DESC LIMIT 20;

-- 模型收入排行
SELECT model_name, SUM(quota) as revenue
FROM logs WHERE created_at > UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 30 DAY))
GROUP BY model_name ORDER BY revenue DESC;
```

### 9.4 收入统计

```sql
-- 今日充值收入
SELECT SUM(money) FROM topups WHERE DATE(FROM_UNIXTIME(create_time)) = CURDATE();

-- 本月充值收入
SELECT SUM(money) FROM topups WHERE MONTH(FROM_UNIXTIME(create_time)) = MONTH(NOW());

-- 充值用户数
SELECT COUNT(DISTINCT user_id) FROM topups WHERE MONTH(FROM_UNIXTIME(create_time)) = MONTH(NOW());
```
